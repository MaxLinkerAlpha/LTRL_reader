# LTRL 标注规范 v1.0

> 基于原书结构和OCR分析结果制定

---

## 一、原书结构分析

### 1.1 标准章节构成

根据OCR分析，LTRL原书每章包含以下部分：

| 部分 | 英文标题 | 中文翻译 | 标注类型 |
|------|---------|---------|---------|
| 1 | **Chapter X** | 第X章 | 【一级标题原文】 |
| 2 | **Vocabulary** | 词汇表 | 【二级标题原文】 |
| 3 | **Vocabulary Notes** | 词汇注释 | 【二级标题原文】 |
| 4 | **Grammar/Morphology** | 语法/形态学 | 【二级标题原文】 |
| 5 | **Observations** | 观察/提示 | 【二级标题原文】 |
| 6 | **Short Readings** | 短篇阅读 | 【二级标题原文】 |
| 7 | **Summary and Synopsis Tables** | 总结与概要表 | 【二级标题原文】 |
| 8 | **Exercises** | 课后练习 | 【二级标题原文】 |

### 1.2 内容类型特征

**词汇表 (Vocabulary)**
- 双栏布局
- 格式：`拉丁词, 属格 词性 英文释义`
- 箭头标记(▶)表示有详细注释
- 示例：`agricola, agricolae m. farmer`

**词汇注释 (Vocabulary Notes)**
- 详细词源解释
- 包含变格表（灰色背景）
- 用法变化说明
- 引文出处（如PLAUTUS, CASINA 690-91）

**短篇阅读 (Short Readings)**
- 拉丁文引文
- 英文翻译
- 词汇注释（逐词解释）
- 出处标注

## 二、标注系统规范

### 2.1 标题层级标注

| 层级 | 标注类型 | 说明 | 示例 |
|------|---------|------|------|
| 一级 | 【一级标题原文】 | 大章节标题 | INTRODUCTION |
| 一级 | 【一级标题译文】 | 大章节标题译文 | 序言 |
| 二级 | 【二级标题原文】 | 章节内子标题 | The Latin Language |
| 二级 | 【二级标题译文】 | 子标题译文 | 拉丁语 |
| 三级 | 【三级标题原文】 | 小节标题 | Alphabet |
| 三级 | 【三级标题译文】 | 小节标题译文 | 字母 |

### 2.2 正文内容标注

| 标注类型 | 判断标准 | 示例 |
|---------|---------|------|
| 【正文原文】 | 1. 包含拉丁词（如anima, fama, deus）<br>2. 英文句子（首字母大写，含空格，以标点结尾）<br>3. **不含中文字符** | The Latin language belongs... |
| 【正文译文】 | 1. **包含中文字符**<br>2. 第2列有【译者：XXX】标记 | 拉丁语属于印欧语系。 |
| 【词汇表】 | 词汇表部分内容，格式为拉丁词+属格+词性+释义 | agricola, agricolae m. farmer |
| 【词汇注释】 | 词汇的详细解释，包含词源 | derives from an Indo-European root... |
| 【原文引文】 | 短篇阅读中的拉丁文原文 | sed etiamne habet nunc Casina gladium? |
| 【引文出处】 | 引文来源标注 | (PLAUTUS, CASINA 690-91) |

### 2.3 表格内容标注

| 标注类型 | 说明 | 示例 |
|---------|------|------|
| 【表格】 | 变格表、变位表等语法表格 | Nominative: deus/dī |
| 【表格标题】 | 表格上方的标题 | MEMORIZE THE FOLLOWING DECLENSION |
| 【表格注释】 | 表格下方的说明 | There is no vocative singular of deus. |

### 2.4 元信息标注

| 标注类型 | 说明 | 格式 |
|---------|------|------|
| 【原书截图】 | 保留截图序号信息 | 【原书截图：33】、【原书截图：34】 |
| 【分工】 | 翻译分工信息 | 序号X，页数X（拆分：N，姓名）... |
| 【译者注释】 | 译者添加的注释 | 注释：1. Faliscan was spoken... |
| 【批注】 | 版本记录、修改标记 | 8.2 改到这里、宗之：... |
| 【页码标记】 | 原书页码 | 【页码：11】、【页码：47】 |

### 2.5 分工信息格式规范

**标准格式：**
```
序号X，页数Y（工作类型：次数，参与者，注释）
```

**工作类型列表：**
- 拆分：文档拆分整理
- 机翻：机器翻译
- 人翻：人工翻译
- 第一次润色（按句）：按句子润色
- 第一次校对：初次校对
- 第二次润色（按意群）：按意群润色
- 第二次校对：二次校对
- 第三次润色：三次润色
- 附加评论/修改建议：评论建议
- 终润（按段落）：最终润色
- 终校：最终校对
- 排版/美化：排版美化

**次数标记：**
- 数字：完成次数（如1, 2, 3）
- N+：表示N次以上（如3+）
- ？：表示未分配/未完成

**示例：**
```
序号34，页数2（拆分：1，Lanx）（机翻：1，Lanx）（人翻：1，Lanx）（第一次润色（按句）：1，Lanx，请先润色其他页数）（第一次校对：？，？）（第二次润色（按意群）：1，Πλατών）（第二次校对：？，？）（终润（按段落）：？，？）（终校：？，？）（排版/美化：？，？）
```

## 三、内容识别规则

### 3.1 原文识别规则（优先级从高到低）

**规则1：拉丁词汇行**
- 匹配模式：`^[a-z]+,`（小写拉丁词开头，逗号后接属格）
- 示例：`agricola, agricolae m. farmer`

**规则2：英文句子**
- 匹配模式：`^[A-Z][a-z]+.*[.!?]$`
- 特征：首字母大写，包含空格，以标点结尾
- 示例：`The Latin language belongs to the Indo-European family of languages.`

**规则3：引文标注**
- 匹配模式：包含括号内的出处
- 示例：`(PLAUTUS, CASINA 690-91)`

**规则4：变格/变位表**
- 匹配模式：连续多行包含格名（Nominative, Genitive等）

### 3.2 译文识别规则

**唯一规则：包含中文字符**
- 匹配模式：`[\u4e00-\u9fff]`
- 辅助验证：第2列有【译者：XXX】标记

### 3.3 特殊情况处理

**情况1：原文跨多行**
- 处理：合并为单行，保留换行标记

**情况2：译文跨多行**
- 处理：合并为单行，同一译者的多行合并

**情况3：原文和译文错位**
- 处理：根据语言特征重新判断

## 四、序号格式规范

### 4.1 原文序号

**完整格式：**
```
【正文原文】序号N 内容
```

**示例：**
```
【正文原文】序号1 The Latin language belongs to...
```

### 4.2 译文序号

**完整格式：**
```
【正文译文】序号N 【译者：姓名】译文内容
```

**示例：**
```
【正文译文】序号1 【译者：机翻】拉丁语属于印欧语系。
```

### 4.3 序号补全规则

对于只写了简单序号的行：
- 自动补全为完整格式
- 保持序号连续性
- 同一原文的多行译文使用相同序号

## 六、Markdown输出格式规范

**期望格式（HTML解析器兼容）：**
```html

```

```html

  </div>

```

```html

  <span class='translator-tag'>译者名</span>
  <span class='translation-text'>译文内容</span>
</div>
```

### 6.3 分工信息格式

```html
<div class='meta-line' data-type='work-assign'>

</div>
```

## 八、附录：常见问题

**Q1：如何判断一行是原文还是译文？**
A1：首先检查是否包含中文字符，含中文为译文，不含中文再判断是否为英文/拉丁文。

**Q2：同一原文有多行译文如何处理？**
A2：保留所有译文，每行独立标注【正文译文】，使用相同序号。

**Q3：分工信息中的注释如何处理？**
A3：保留在括号内，如`（第一次润色（按句）：1，Lanx，请先润色其他页数）`

**Q4：截图序号如何标准化？**
A4：从【截图序号：33】改为【原书截图：33】，保留序号信息。
